T5 Problem Solving Blog

What was the problem?

Syntax errors!! I would get them constantly, but find it so difficult to fix them.

What problem solving techniques did you use?

I broke down the error, figuring out exactly where abouts in my code I had gone wrong, then read over it to make sure I had everything correct. I would try adding things or taking things away until the error was fixed.

How did you feel throughout the process?

Frustrated, useless and upset when it didnt go right - but when it did work I felt on top of the world, and I felt like I could do anything!

What did you learn?

Be patient with yourself, take time to slowly and carefully go over these things, the more you rush the harder it is to pick up on these errors, and its generally why you make mistakes in the first place - if you're always rushing.

Pseudocode

It is a text-based method for preparing the code that you want to create. It's written in English, but its a useful tool to give you step by step to give you the result you want. I use this form of problem solving because it helps prevent creating a problem in the first place.

Trying something

The easiest way of problem solving - if it doesn't work, go back and try something different.

.map()

When the .map function is called on an array it creates a new array made up of the results of a chosen function on each element of the parent array.

.filter()

Creates a new array of all the elements in the parent array which comply to the conditions set out in the function within the filter parameters.

.reduce()

Reduces the values of an array to a single value. It uses two parameters and then a function to return the single value. This can also be used to flatten an array of arrays into a single array.
